Вся командная переписка и так в Telegram – значит и задачи удобнее вести там же. **taskbara** – примитивный, но удобный трекер: ставишь задачу прямо в чате, у каждой короткий git-подобный хэш (`4951cd3`), два статуса (открыта / сделано), комментарии и редактирование. Без дашбордов, дедлайнов и лишних кликов.

**Особенности:**

- **Хэши как в git** – каждая задача получает короткий копируемый идентификатор `4951cd3`
- **Два статуса** – `открыта` / `сделано`, ничего лишнего
- **Работает в любом чате** – добавь бота в группу и пользуйся
- **Проверка участников** – `@упоминания` сверяются с реальными участниками чата, регистр ника исправляется автоматически
- **Логи в файл и консоль** – с таймстампом и уровнем (`INFO` / `WARNING` / `ERROR`)

## Быстрый старт

> [!NOTE]
> Требуется Python **3.11+**

### 1. Создай бота

Напиши [@BotFather](https://t.me/BotFather) → `/newbot`, следуй инструкции и получи **токен**.

> [!IMPORTANT]
> Отключи приватный режим, иначе бот не увидит участников чата:
> `/setprivacy` → выбери бота → **Disable**.

### 2. Установка и запуск

```bash
git clone https://github.com/Aleksooo/taskbara.git
cd taskbara

pip install uv
uv sync
```

Запусти бота:

```bash
uv run taskbara
```

При первом запуске бот попросит вставить `BOT_TOKEN` прямо в консоли и сохранит его в `.env` – дальше токен подхватывается автоматически.

### 3. Пользуйся

Добавь бота в рабочий чат и отправь `/help`.

## Команды

| Команда | Что делает |
|---|---|
| `/addtask @from to @to` + текст | создать задачу: `@from` – постановщик, `@to` – исполнитель |
| `/addtask @to` + текст | то же, но постановщик – ты |
| `/tasks @user` | задачи, поставленные `@user` (сгруппированы по статусу и дате) |
| `/tasks` | твои задачи |
| `/task <хэш>` | полный текст задачи и комментарии |
| `/edit <хэш>` + текст | заменить текст задачи |
| `/done <хэш>` | пометить как сделано |
| `/reopen <хэш>` | снова открыть |
| `/addcomment <хэш>` + текст | добавить комментарий |
| `/help` | список всех команд |

Текст задачи или комментария можно писать сразу после команды или на следующей строке.

## Конфигурация

Настройки читаются из `.env` (создаётся при первом запуске):

| Переменная | По умолчанию | Описание |
|---|---|---|
| `BOT_TOKEN` | – | токен от @BotFather (обязателен) |
| `DB_PATH` | `taskbara.db` | путь к файлу SQLite |
| `LOG_FILE` | `taskbara.log` | путь к лог-файлу |
| `LOG_LEVEL` | `INFO` | уровень логирования |

## Структура проекта

```
src/taskbara/
├── bot.py            # точка входа: config → logging → БД → polling
├── config.py         # загрузка .env + CLI-запрос токена
├── logging_setup.py  # логи в консоль и файл
├── db.py             # схема и async-запросы (aiosqlite)
├── hashing.py        # генерация уникального хэша задачи
├── parsing.py        # разбор команд и аргументов
├── formatting.py     # сборка текстов ответов
└── handlers.py       # aiogram-роутер: хендлеры команд
tests/                # unit-тесты parsing / formatting / db / hashing
```
